# C/S常用协议

## HTTP 

- **名称** Hyper text Transfer Protocal，超文本传输协议
- **端口号** 80
- **功能** 主要是服务于 Web 服务，进行数据传输
- **连接类型**
    - **非持续连接** 一次只能传输一个对象
    - **持续链接** 一次传输多个对象 http1.1
        - **非流水线连接** 等待**请求响应完成后**再发出下一个请求
        - **流水线连接** 在持续连接上**连续发送多个请求**而**不用等待**每个请求的响应
- **报文**
    - **类型**：请求（方法），相应（代码：如200和404）
    - [**格式**](https://blog.csdn.net/sunyctf/article/details/129002163)
        - **请求行 / 状态行**（分别对应于请求报文和响应报文）
        - **首部行**
        - **结束行**
    - **实体体**
- **cookies**
- **cache**

## FTP

- **端口号**：控制部分为21，传输部分为20
- **带外传输**：控制与传输相对分离

## SMTP / POP3 / IMAP

**SMTP**：简单邮件传输协议 ( Simple Mail Transfer Protocal )，从发件人到邮件服务器和服务器之间的协议是SMTP协议，SMTP的端口号为25

**POP3、HTTP 和 IMAP**：接受邮件协议，收件人从邮件服务器拉取邮件的协议是这些协议

POP3 与 IMAP 协议的区别在于 IMAP 提供了更多的功能，如：邮件分组，像文件一样管理

对比：**SMTP** 和 **HTTP**，**POP3** 和 **IMAP**

**邮件发送的六个步骤**

1. 发件人在用户代理撰写邮件
2. 用户代理发送到邮件服务器，邮件放在报文队列中
3. SMTP的客户端打开到收件人邮件服务器的TCP连接
4. SMTP客户端通过TCP连接发送邮件
5. 收件人的邮件服务器将邮件放到收件人的邮箱中
6. 收件人的用户代理接收邮件

报文类型、交互和格式

**邮件只能承载以 ASCII 编码的字符，通过 Base64 等编码扩展内容范围，使之能够承载包括可执行文件在内的所有文件**

## DNS

- **名称**：**Domain Name System 域名系统**
- **端口号**：**53**
- **传输层协议：UDP**
- **两部分**
    1. 查询报文
    2. 应答报文
- **三功能**
    1. **域名到 IP 地址的转换**（基本功能）
    2. **别名**
    3. **负载均衡**
- **怎样实现**
    - **层次架构**
        - **根**DNS服务器
        - **顶级**DNS服务器
        - **权威**DNS服务器
    - **查找方式**
        - **递归**查找：本机向本地域名服务器发出一次查询请求，就静待最终的结果。如果**本地域名服务器无法解析**，自己会以DNS**客户机的身份**向**其它域名服务器**查询，直到得到最终的IP地址告诉**最终客户机**
        - **迭代**查找：上一个服务器查找后如果没找到，那么服务器会**返回下一个服务器的IP地址**，然后客户机自己再去查找——**亲历亲为**
- 类型和格式
    - A、CNAME、NS、MX、AAAA 等记录类型
    - 记录的格式（五个字段）
        1. **记录类型**
        2. **TTL** （Time To Live，在服务器中的存活时间）
        3. **域名**
        4. **IP 地址**
        5. **协议类型**（大多是IN——Internet）
    - 报文格式
