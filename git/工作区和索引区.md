![[Pasted image 20250731144830.png]]
可以看到 file2.txt 消失了, 这说明 `index` 文件保存了**当下跟踪的文件的文件名**, 再从那些乱码中来看, 应该还有**哈希码**.

> [!important]
> index 中存储了一些信息
> 
> 1. 文件元数据 (metadata) : **文件名**, **文件种类** (二进制还是文本), **文件权限**和**最后修改时间**等
> 2. 文件快照: 这个文件对应的 **`blob` 对象的哈希值**
> 3. 目录树信息, 记录**文件与目录的关系**
> 4. 冲突标记: 合并**冲突**

## git 怎么知道文件发生了改动

### `git status` 命令

> [!info]
> 通过 `git status` 命令可以获取当前工作区下文件的状态

#### 新文件感知 `untracked`

### `git ls-files` 命令

> [!info]
> 通过 `git ls-files` 可以查看当前追踪的所有文件, 添加一个 `-s` 参数能够显示更多信息

![[Pasted image 20250731151522.png]]

> [!important]
> 有没有发现和上面的输出的区别? 没错, **哈希值变了**, 在[[git add#使用 `-t` 参数查看文件类型]]的第二个 info 中提到了 git **哈希计算的公式**. 可以看到这个公式融合了**文件内容**和**文件长度**. 哈希函数具有**防碰撞特性**, 只需要一位改变, 计算出来的哈希值差别就会巨大, 完全不可测量. git 通过这样的哈希值来判断一个文件是否和以前的版本一致.
> 
> 这是 git 判断两个文件是否相同的重要方法
