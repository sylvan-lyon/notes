# `git add` 干了什么 ?


>[!question]
> 我们都会在提交前运行 `git add` 命令, 将做出的更改交给 git, 这个 git add 干了什么? 为什么不能直接 commit?


1. 首先创建一个文件, 就叫他 `hello.txt` 吧, 内容就是 `hello git`.
2. 运行 `git add` 将文件做出的更改从**工作区**添加到**暂存区**( [[工作区和索引区]], 此处不需要了解这两个术语 )
3. 分析一下下

### 查看运行后的目录结构

![[Pasted image 20250731111125.png]]

可以看到 `.git/objects` 目录下多了一些东西, 一个 `index` 文件, 一个文件名杂乱无章的文件 和 一个目录名不明所以的文件夹.

> [!info]
> 如果接触过编程的话, 应该一眼就能看出这是一个**哈希值**, 实际上 git 使用 **SHA1** 进行哈希, 具有良好的性能.

>[!info] 
> 各位会发现在**哈希值**和**文件大小**和我不一样, 这是因为我使用的是 [nushell](https://www.nushell.sh/zh-CN/), 这个 nushell 的 echo 实现和 [posix shell](https://baike.baidu.com/item/%E5%8F%AF%E7%A7%BB%E6%A4%8D%E6%93%8D%E4%BD%9C%E7%B3%BB%E7%BB%9F%E6%8E%A5%E5%8F%A3/12718298) ( 如 bash, zsh ) 不一样. [posix shell](https://baike.baidu.com/item/%E5%8F%AF%E7%A7%BB%E6%A4%8D%E6%93%8D%E4%BD%9C%E7%B3%BB%E7%BB%9F%E6%8E%A5%E5%8F%A3/12718298) 会在 echo 的内容后面追加一个换行符 `\n`, 而 [nushell](https://www.nushell.sh/zh-CN/) 不会, 导致内容不一样

## 使用 `git cat-file` 查看文件信息

#### 使用 `-t` 参数查看文件类型

![[Pasted image 20250731111723.png]]

可以看到输出了一个 **`blob`**, 这个**文件类型用于存储文件内容** ( 仅文件内容! )

> [!info]
> `f90e9c` 是完整哈希的**前 6 个字符**, 将**文件夹的名字和文件名拼接在一起才是完整的哈希值** ( 各位应该在很多包管理器或者代码仓库见过这种形式, 比如 maven, crates 等 )
> 
> 你完全可以使用前 10 位 8 位来表示这个文件对象, **只需要能够唯一标识这个文件就可以**, 在这个例子中, 由于文件对象很少, 你甚至可以只用 `f09e` 四个字符来标识这个文件对象 ( **git 最少需要四位表示, 通常对于我们人类, 最好是取六位** )

> [!important]
> 注意, 此处我使用**文件对象**来描述这个文件名乱码七糟的文件, 我们将很快认识到**文件到文件对象是多对一的关系**

如果我们将另一个文件名不同但是内容相同的文件添加到[索引区](工作区和索引区.md)中, 我们将不会看到 `objects` 文件夹中多了些文件 / 文件夹

除了多了一个 **tmp.txt** 外并无不同

在此声明, 这个哈希值混合了 **文件对象的类型**, 不然如何查看文件记录的类型?

> [!info]
> git 使用的盐: `echo "blob ${文件大小}\0${文件内容}" | shasum` 这里的 `shasum` 是假设的计算哈希值的工具. 例如 我的这个 9 字节长的文件的哈希字符串为: `blob 9\0hello git`
