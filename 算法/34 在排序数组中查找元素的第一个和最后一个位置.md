# [题目](https://leetcode.cn/problems/find-first-and-last-position-of-element-in-sorted-array/description/)

## 分析

由于题目说数组是**有序**的，那么我们可以很轻松的得到对应元素的开始。而最后一个元素的为止呢？我们可以通过寻找 `target + 1` 的位置，并将 `target + 1` 的开始减一得到最后一个 `target` 的位置

---

## 实现


>[!NOTE]
>直接使用
>$$
>\text{avg}(a, b) = \frac{(a + b)}{2}
>$$
>容易导致溢出，应当使用优化公式
>$$
>\text{avg}(a, b) = a - \frac{(a - b)}{2}
>$$ 


>[!IMPORTANT]
>注意这里的区间开闭，非常重要！！！这关系着会不会死循环。
>
>重要原则是：
>- 使用**闭区间**就把**折半后的范围**归约到**闭区间的形式**
>-  使用**开区间**就把**折半后的范围**归约到**开区间的形式**
>- 使用**半开区间**就把**折半后的范围**归约到**半开区间的形式**
>
>也就是说**原来是什么**，**折半后的形式**就是什么，然后进行 `left` `right` 的赋值
>
>注意 `mid` 始终取 `left` `right` 的平均值


首先，二分的精髓在于每次迭代都能够**缩小一半**的**查找范围**，这个查找范围尤其重要！
### 闭区间二分

初始化三个指针：`left = 0`， `right = length`， `mid = avg(left, right)`

#### 查找范围

查找范围应当是：$[\text{left}, \text{right}]$，对 `nums[mid]` 与 `target` 的大小关系进行讨论：

- `nums[mid]` $\geq$ `target` : 这意味着从 $[\text{mid}, \text{right}]$ 的区间都是大于等于 `target` 的，范围折半到 $[\text{left}, \text{mid})$， 也就是 $[\text{left}, \text{mid} - 1]$，`right` 就应当赋值为 `mid - 1`
- `nums[mid]` $<$ `target` : 这意味着从 $[\text{left}, \text{mid}]$ 的区间都是小于 `target` 的，范围折半到 $(\text{mid}, \text{right}]$ ， 也就是 $[\text{mid} + 1, \text{right}]$，`left` 就应当赋值为 `mid + 1`

#### 代码

```go
// lowerBound 返回最小的满足 nums[i] >= target 的下标 i
// 如果数组为空，或者所有数都 < target，则返回 len(nums)
// 要求 nums 是非递减的，即 nums[i] <= nums[i + 1]
func lowerBound(nums []int, target int) int {
    left, right := 0, len(nums)-1 // 闭区间 [left, right]

    for left <= right { // 区间不为空
        // 循环不变量：
        // nums[left-1] < target
        // nums[right+1] >= target
        // 每次循环给 [left, right] 染色
        mid := left + (right-left)/2
        if nums[mid] >= target {
            // 可得出 [mid, right] 都是大于等于 target 的
            right = mid - 1 // 范围缩小到 [left, mid-1]
        } else {
            // 可得出 [left, mid] 都是小于 target 的
            left = mid + 1 // 范围缩小到 [mid+1, right]
        }
    }

    // 循环结束后 left = right+1
    // 此时 nums[left-1] < target 而 nums[left] = nums[right+1] >= target
    // 所以 left 就是第一个 >= target 的元素下标
    return left
}
```

### 开区间二分

开区间写法将查找范围表示为 `(left, right)`，即不包含两端点。

#### 初始化

- `left = -1`，`right = len(nums)`  
  区间 `(left, right)` 最初包含了所有有效下标 `[0, len(nums)-1]`，因为 `left` 和 `right` 本身不在查找范围内。

#### 循环条件

- `left + 1 < right`  
  当区间内至少有一个整数时继续循环。

#### 区间缩小时的原则

保持新区间仍然是开区间 `(left, right)`，通过移动 `left` 或 `right` 来缩小范围。

- 如果 `nums[mid] >= target`，说明 `mid` 及右侧均 ≥ target，那么 `right` 应移动到 `mid`（因为 `mid` 可能正是第一个 ≥ target 的位置，且开区间右端点不包括它）。
- 如果 `nums[mid] < target`，说明 `mid` 及左侧均 < target，那么 `left` 应移动到 `mid`（因为 `mid` 不可能是第一个 ≥ target 的位置，且开区间左端点不包括它）。

#### 循环不变量

- `nums[left] < target`（若 `left` 在合法范围内）
- `nums[right] >= target`（若 `right` 在合法范围内）

#### 最终返回值

循环结束时 `left + 1 == right`，此时 `left` 是最后一个 < target 的位置，`right` 是第一个 ≥ target 的位置。所以返回 `right` 即为所求。

#### 代码实现

```go
func lowerBound(nums []int, target int) int {
    left, right := -1, len(nums) // 开区间 (left, right)

    for left+1 < right { // 区间内至少有一个整数
        // 循环不变量：
        // nums[left] < target （若 left >= 0）
        // nums[right] >= target （若 right < len(nums)）
        mid := left + (right-left)/2
        if nums[mid] >= target {
            right = mid // 范围缩小到 (left, mid)
        } else {
            left = mid // 范围缩小到 (mid, right)
        }
    }
    // 循环结束后 left+1 == right
    // 此时 nums[left] < target 且 nums[right] >= target
    // 所以 right 就是第一个 >= target 的元素下标
    return right
}
```

### 半开区间二分（左闭右开）

半开区间写法将查找范围表示为 `[left, right)`，包含左端点，不包含右端点。

#### 初始化

- `left = 0`，`right = len(nums)`  
  区间 `[left, right)` 包含了所有有效下标 `[0, len(nums)-1]`。

#### 循环条件

- `left < right`  
  当区间非空时继续循环。

#### 区间缩小时的原则

保持新区间仍然是左闭右开 `[left, right)`。

- 如果 `nums[mid] >= target`，说明 `mid` 及右侧均 ≥ target，那么 `right` 应移动到 `mid`（因为 `right` 是开端点，不包含 `mid`）。
- 如果 `nums[mid] < target`，说明 `mid` 及左侧均 < target，那么 `left` 应移动到 `mid+1`（因为 `mid` 及之前的都不可能是第一个 ≥ target 的位置，新的左端点从 `mid+1` 开始）。

#### 循环不变量

- `nums[left-1] < target`（若 `left > 0`）
- `nums[right] >= target`（若 `right < len(nums)`）

#### 最终返回值

循环结束时 `left == right`，此时 `left` 是第一个 ≥ target 的位置，直接返回 `left` 即可。

#### 代码实现

```go
func lowerBound(nums []int, target int) int {
    left, right := 0, len(nums) // 左闭右开 [left, right)

    for left < right { // 区间非空
        // 循环不变量：
        // nums[left-1] < target （若 left > 0）
        // nums[right] >= target （若 right < len(nums)）
        mid := left + (right-left)/2
        if nums[mid] >= target {
            right = mid // 范围缩小到 [left, mid)
        } else {
            left = mid + 1 // 范围缩小到 [mid+1, right)
        }
    }
    // 循环结束后 left == right
    // 此时 nums[left-1] < target 且 nums[left] >= target
    // 所以 left 就是第一个 >= target 的元素下标
    return left
}
```

### 总结

三种区间写法的核心思想一致，只是对边界处理略有不同，但都能正确实现 `lowerBound` 函数。无论使用哪种，都可以通过：

```go
start := lowerBound(nums, target)
end := lowerBound(nums, target+1) - 1
```

得到目标值的第一个和最后一个位置（若 `start == len(nums) || nums[start] != target` 则说明不存在）。
