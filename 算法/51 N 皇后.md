# [题目](https://leetcode.cn/problems/n-queens/description/)

## 分析

N 皇后问题要求在 `n × n` 的棋盘上放置 `n` 个皇后，使得它们互不攻击（即任意两个皇后不在同一行、同一列或同一对角线上）。

一个重要结论是：**如果在一个 `n × n` 的棋盘上放置了 `n` 个皇后且它们互不攻击，那么每行、每列都必须恰好有一个皇后**。

> [!NOTE]
> 证明：
> 1. 不可能有一行或一列包含多于一个皇后，否则这两个皇后会互相攻击。
> 2. 不可能有一行或一列没有皇后，否则 `n` 个皇后必须分布在剩下的 `n-1` 行（或列）中，根据抽屉原理，必然有一行（或列）包含至少两个皇后，与1矛盾。

因此，我们可以将问题转化为：**在每一行放置一个皇后，并记录其列坐标**。搜索时只需枚举每一行的列位置，并检查是否与之前放置的皇后冲突。

冲突条件为：
- 同一列
- 同一主对角线（行差等于列差）
- 同一副对角线（行差等于列差的相反数）

## 实现

### 回溯法 (基本实现)

使用深度优先搜索（回溯）逐行放置皇后。用数组 `path` 记录已放置的皇后位置：`path[y] = x` 表示第 `y` 行的皇后位于第 `x` 列（`y` 从 0 开始）。

#### 代码

```go
func solveNQueens(n int) [][]string {
    // path 记录每行皇后所在的列下标
    // 物理意义：path[y] = x 表示纵坐标为 y 的皇后的横坐标为 x
    path := make([]int, 0, n)
    answer := [][]string{}

    var backtrack func(int)
    backtrack = func(remain int) {
        if len(path) == n {
            // 构造一个合法棋盘
            tmp := make([]string, 0, n)
            for _, x := range path {
                row := make([]byte, n)
                for i := range n {
                    if i == x {
                        row[i] = 'Q'
                    } else {
                        row[i] = '.'
                    }
                }
                tmp = append(tmp, string(row))
            }
            answer = append(answer, tmp)
            return
        }

        // 尝试在当前行（第 len(path) 行）的每一列放置皇后
        for x := range n {
            if valid(path, x) {
                path = append(path, x)
                backtrack(remain - 1)
                path = path[:len(path)-1]
            }
        }
    }

    backtrack(n)
    return answer
}

// valid 检查在当前行（即第 len(path) 行）的 x 列放置皇后是否合法
func valid(path []int, newX int) bool {
    newY := len(path)               // 新皇后所在的行
    for y, x := range path {
        dx, dy := newX - x, newY - y
        // dy 不可能为 0（因为每行只有一个皇后）
        // 检查是否在同一列 (dx == 0) 或同一对角线 (dx == dy 或 dx == -dy)
        if dx == dy || dy == -dx || dx == 0 {
            return false
        }
    }
    return true
}
```

#### 解释

1. **搜索顺序**：从上到下逐行放置皇后，每行选择一个列位置。
2. **合法性检查**：`valid` 函数将新皇后与之前每一行的皇后进行比较，确保不同列、不同对角线。
3. **构造答案**：当 `path` 长度达到 `n` 时，说明已经成功放置了 `n` 个皇后，此时根据 `path` 数组构造棋盘字符串（`Q` 表示皇后，`.` 表示空位）。

### 优化：使用集合记录已占用的列和对角线

上述实现每次检查都需要遍历之前的所有皇后，时间复杂度为 `O(n)`。可以用三个布尔数组（或集合）分别记录**已占用的列**、**已占用的主对角线**、**已占用的副对角线**，将检查时间降至 `O(1)`。

主对角线的下标为 `y - x`（范围 `[-(n-1), n-1]`），副对角线的下标为 `y + x`（范围 `[0, 2n-2]`）。可以用偏移量将其映射到非负下标。

#### 优化代码

```go
func solveNQueens(n int) [][]string {
    colUsed := make([]bool, n)          // 列占用标记
    mainDiag := make([]bool, 2*n-1)     // 主对角线占用标记，下标 = y - x + n - 1
    antiDiag := make([]bool, 2*n-1)     // 副对角线占用标记，下标 = y + x
    path := make([]int, 0, n)
    answer := [][]string{}

    var backtrack func(int)
    backtrack = func(row int) {
        if row == n {
            board := make([]string, n)
            for y, x := range path {
                b := make([]byte, n)
                for i := range b {
                    if i == x {
                        b[i] = 'Q'
                    } else {
                        b[i] = '.'
                    }
                }
                board[y] = string(b)
            }
            answer = append(answer, board)
            return
        }

        for x := 0; x < n; x++ {
            // 计算对角线下标
            mainIdx := row - x + n - 1
            antiIdx := row + x
            if !colUsed[x] && !mainDiag[mainIdx] && !antiDiag[antiIdx] {
                // 放置皇后
                colUsed[x] = true
                mainDiag[mainIdx] = true
                antiDiag[antiIdx] = true
                path = append(path, x)

                backtrack(row + 1)

                // 回溯
                path = path[:len(path)-1]
                colUsed[x] = false
                mainDiag[mainIdx] = false
                antiDiag[antiIdx] = false
            }
        }
    }

    backtrack(0)
    return answer
}
```

## 时空复杂度分析

### 基础回溯法
- **时间复杂度**：最坏情况下需要尝试所有可能的放置方式。搜索树的最大深度为 `n`，每层分支数最多为 `n`，且每次检查合法性需要 `O(n)` 时间，因此总时间复杂度为 `O(n! * n)`（实际上远小于 `n!`，因为很多分支会提前被剪枝）。
- **空间复杂度**：递归栈深度为 `O(n)`，`path` 数组占用 `O(n)`，构造答案的临时空间不计入，总体为 `O(n)`。

### 优化版本（使用集合记录）
- **时间复杂度**：每层分支数最多为 `n`，但每次检查合法性只需 `O(1)`，因此总时间复杂度约为 `O(n!)`（依然受剪枝影响）。
- **空间复杂度**：需要三个布尔数组，大小分别为 `n`、`2n-1`、`2n-1`，以及递归栈和 `path` 数组，总体为 `O(n)`。

## 总结

N 皇后问题是回溯算法的经典例题。关键点在于利用“每行必须放一个皇后”将问题转化为在每一行选择列位置，并通过剪枝排除冲突的放置。优化版本使用额外的数组记录列和对角线的占用情况，将每次检查的时间降为常数，提高了搜索效率。
